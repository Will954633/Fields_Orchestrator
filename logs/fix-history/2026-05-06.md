## [APPRAISAL-ROOM-CARDS] PVD/consumer shape mismatch — only 3 of 6 room cards rendered — 21:18 AEST
**Symptom:** Page 5 ("Your Property Through Our Eyes") of the seller appraisal report rendered only 3 cards (Kitchen, Exterior, Outdoor). Bathrooms, Living Areas, and Master Bedroom were silently skipped on every property.

**Root cause:** Pure downstream consumer bug. The Ollama enrichment schema in `enrich_for_sale_batch.py` emits `bathrooms`, `living_areas`, and `bedrooms` as **arrays** (one entry per visible room, with the master bedroom tagged inside `bedrooms[]` via `bedroom_label: "master"` — there is no separate `master_bedroom` field). The downstream consumer `build_room_assessments()` in `scripts/generate_appraisal_report.py` was written for a flat-dict shape: it called `pvd.get("bathrooms")`, saw a list, and bailed via `isinstance(data, list): continue`. Master Bedroom was never present at top level so it always fell through. Sampled 60 properties across robina/burleigh_waters/varsity_lakes/merrimac → 100% array shape for bathrooms/living_areas/bedrooms when PVD existed, 100% missing for `master_bedroom`. Upstream is healthy and unchanged.

Two secondary bugs were also fixed: detail-field names in the mapping referenced fields that don't exist on the schema (`tiling_condition` → real is `tile_condition`; `flooring_material` → `flooring_type`; `ensuite_quality` → `ensuite`), and the boolean detail formatter rendered `True` literally.

**Fix:**
- Added `_collapse_room_array(arr)` — picks the highest `quality_score` (fallback `condition_score`, then first visible) entry from a list-shape room. Lets a single card represent the strongest room of its type, consistent with the existing "most generous reasonable read" semantics in `_derive_room_score`.
- Added `_extract_master_bedroom(pvd)` — finds the entry in `pvd['bedrooms']` with `bedroom_label == "master"`. Returns None if absent (so the card is skipped rather than guessed).
- Patched `build_room_assessments()`: master_bedroom now falls back to the bedrooms[] extraction; arrays are collapsed before scoring; legacy dict shape still works (backwards-compatible).
- Updated `mapping` detail field names to match the real schema (`tile_condition`, `vanity_style`, `flooring_type`, `ceiling_height`, `walk_in_robe`, `ensuite`).
- Boolean detail values now render as "Yes" instead of `True` (False is already filtered out).
- Verified by re-rendering `output/seller_reports/2026-05-06_59-pintail-crescent_reference_v2.pdf` via `--reuse-editorial` (no AI cost): all 6 cards now render — Kitchen 8/10, Bathrooms 8/10, Living Areas 9/10, Master Bedroom 9/10, Exterior 7/10, Outdoor 9/10.
