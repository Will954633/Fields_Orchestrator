# Fix History — 2026-05-18

## [COVERAGE-AUDIT-001] Address-level coverage audit script + bungled-upload verification — 15:45 AEST

**Symptom:** No tooling existed to audit per-address data coverage across the cadastral baseline. Will spot-checked 1 Glenside Dr Robina and found no photo. Also unverified status on the 2026-05-15 apr01 mongodump merge — did the recovered image URLs actually land on the right records and are they live?

**Root cause:** Coverage was tracked per-listing in the ops dashboard (enrichment % of active listings), not per-cadastral-address. Recovery from the IMAGE-OUTAGE-001 blob shutdown was tracked in fix logs but never measured against the cadastral baseline.

**Fix:** New `scripts/address_coverage_audit.py` joins every cadastral record in `Gold_Coast.{robina, burleigh_waters, varsity_lakes}` against every data dimension:
- listing presence (ever listed / active / sold)
- image sources: `scraped_data_v2`, `scraped_data_apr01_recovered`, Phase-2 sidecars, `property_images*`, `scraped_property_images`
- floor plans, photo analysis, floor-plan analysis, satellite analysis
- valuation (reconciled + CatBoost + Domain)

Three modes: `--summary` (per-suburb markdown table), `--csv` (per-address rows), `--address "..."` (single lookup). Optional `--check-urls N` does live HEAD requests against random sample.

- **apr01 mongodump merge succeeded:** 17,393 records across target suburbs gained recovered images. 8/8 sampled URLs live (Domain CDN, not the dead Azure blob).
- **v2 re-scrape uneven:** Robina ran with 67% failure rate (4,981 failed / 7,451 attempted), Burleigh Waters never ran v2, Varsity Lakes barely started (~11% coverage).
- **Phase-2 sidecars added zero rows** to target suburbs — Phase 1 `_id` match captured everything matchable.
- **110 currently-listed properties (47+20+43) still have ZERO image** across all sources — the actual gap to close.
- 1 Glenside Dr Robina specifically: 40,900 m² cadastral lot, never listed, v2 scrape was attempted and failed. Expected — no listing means no image source.
