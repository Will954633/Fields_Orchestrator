# Fix History — 2026-05-26

## [CLAUDE-AUTH-LEAK] Interactive shells leaking ANTHROPIC_API_KEY into `claude` CLI — 08:48 AEST
**Symptom:** Claude Code OAuth subscription session intermittently failing after laptop sleep/reconnect; "paste the code" OAuth flow producing 400s; suspicion that `claude` CLI in terminals was using API-key billing instead of the Max subscription.
**Root cause:** Two compounding issues found inside the VM:
1. `.env:125` had an uncommented `ANTHROPIC_API_KEY=...` (re-added after a prior "cleanup" — line 116 has the commented `_DISABLED_FOR_SUBSCRIPTION` variant but a second uncommented export had crept back in).
2. `/home/projects/.bashrc:119` auto-sources the entire `.env` into every interactive shell, so every code-server integrated terminal inherited `ANTHROPIC_API_KEY` and any `claude` CLI launched from it preferred API-key auth over `~/.claude.json` OAuth.
The separate browser ERR_CONNECTION_REFUSED on the OAuth callback is a different problem (code-server bound to 127.0.0.1:8080 via reverse proxy at `vm.fieldsestate.com.au`, no auto port-forward for the extension's random callback port — requires SSH tunnel or local VS Code + Remote-SSH).
**Fix:** Added `unset ANTHROPIC_API_KEY` immediately after the `.env` auto-source in `~/.bashrc` so interactive shells no longer export the key. Variable remains in `.env` for scripts that explicitly `source` it (editorial generation etc). Extension-host process was already clean (verified via `/proc/<pid>/environ`).
**Files:** `/home/projects/.bashrc`
**Recurrence:** Second time this surface (prior cleanup removed the key but it was re-added to `.env` and the bashrc auto-source was never addressed).
