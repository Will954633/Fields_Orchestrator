## [CHART-SCHEDULE] All market metrics charts moved to monthly-on-1st update cycle — 10:40 AEST
**Symptom:** Charts updated daily/weekly while commentary only updated monthly, causing data-commentary drift.
**Root cause:** Precompute crons ran daily (median price at 22:27, DOM at 22:35) and SQM scraper ran weekly (Sunday 04:00). Charts would shift mid-month after commentary was written.
**Fix:** Changed all 5 chart data pipelines to run on the 1st of each month only, at 05:00–05:32 AEST. Sequence: previous night pipeline at 20:30 populates raw data → 05:00 indexed prices → 05:08 DOM+cycle → 05:16 volume+turnover → 05:24 absorption snapshots → 05:32 SQM scrape. All data fresh by 05:32, ready for 08:00 commentary session.
**Files:** crontab (3 schedule changes: precompute_indexed daily→1st 05:00, precompute_market_charts dom daily→1st 05:08, sqm_asking weekly→1st 05:32)

## [ABSORPTION-SNAPSHOT] Monthly absorption rate snapshots — chart updates 1st only — 10:25 AEST
**Symptom:** Absorption rate chart updated daily (rolling 30-day window) causing drift between chart values and monthly commentary.
**Root cause:** market-insights.mjs computed absorption rate live on every API request.
**Fix:** Added monthly snapshot system — on the 1st (or first request of a new month), the function computes fresh values and stores to `system_monitor.absorption_rate_snapshots`. All subsequent requests that month serve the cached snapshot. Added `scripts/refresh_absorption_snapshots.py` to delete snapshots on the 1st at 22:45 AEST (after pipeline), triggering recomputation on next request.
**Files:** netlify/functions/market-insights.mjs, scripts/refresh_absorption_snapshots.py, crontab
