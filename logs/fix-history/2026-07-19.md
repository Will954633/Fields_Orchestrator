## [BRAIN2-504] Nightly PostHog builders die silently on 504 → stale Brain 2 — 19:50 AEST
**Symptom:** Brain 2 "did it update last night?" check (Jul 18): `ad_attribution_build` and `organic_journey_build` both crashed with `HTTP Error 504: Gateway Timeout` on their heaviest HogQL queries → `ad_downstream`, `organic_journeys`, `all_conversions`, `organic_landing_affinity`, `ai_referral_signal` silently stale by a day. No alert; only visible by reading the log.
**Root cause:** each builder's `hog()` did a single `urllib.request.urlopen(timeout=120)` with no retry. PostHog returns transient 504s on the largest scans (grows with data volume). The nightly chain is `;`-joined so one crash doesn't stop the others, but also nobody is notified — a failed builder just leaves last night's data in place.
**Fix:** New `scripts/brain2/brain2_util.py` — `hog_retry()` (3 retries, exp backoff 5/15/45s, retries 5xx + timeout + URLError, re-raises 4xx immediately) and `alert_failure()` (Telegram ping naming the builder + re-run command). Rewired all three chain builders' `hog()` to `hog_retry` and wrapped each `main()` in try/except→`alert_failure`→re-raise. Recovered Jul 18's stale data by re-running (504 was transient). Verified: compile + import + live `hog_retry` query all OK; both rebuilt builders run clean.
**Files:** scripts/brain2/brain2_util.py (new), scripts/brain2/ad_attribution_build.py, scripts/brain2/ad_behaviour_build.py, scripts/brain2/organic_journey_build.py
**Recurrence:** First (504 handling); relates to prior BRAIN2 HogQL LIMIT-100 gotcha (same hog() helper).
