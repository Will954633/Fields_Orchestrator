## [PHOTO-ANALYSIS-TOKEN-LIMIT] Fix step 105 photo analysis 100% failure rate — 18:25 AEST

**Symptom:** Step 105 (Photo Analysis) succeeded 0 out of 93 properties in last pipeline run. Error: `Expecting value: line 1 column 1 (char 0)` — empty JSON response from GPT API. All properties across Robina, Varsity Lakes, and Burleigh Waters affected.
**Root cause:** `gpt-5-nano-2025-08-07` uses reasoning tokens that count against `max_completion_tokens`. **Fix:** Increased `max_completion_tokens` from 1000 to 4000 in `ollama_client_single_image.py`. Tested and confirmed full JSON responses.
**Files:** `Property_Data_Scraping/.../Ollama_Property_Analysis/ollama_client_single_image.py`
**Recurrence:** First occurrence — caused by model upgrade to gpt-5-nano which introduced reasoning tokens.

## [CEO-STEP116-RETRY] Improve Data Quality Validator Cosmos retry logic — 08:34 AEST

**Symptom:** Step 116 failed after 3 attempts with Cosmos DB 429 (TooManyRequests) mid-cursor on burleigh_heads. Watchdog classified as UNKNOWN and escalated.
**Root cause:** `safe_find()` retried from scratch on 429, losing all accumulated docs. Only 5 retries with fixed 5s×attempt backoff — insufficient when Cosmos is under sustained load from earlier pipeline steps.

## [CEO-COST-ROLLUP] Fix cost rollup showing $0 ad spend after 2026-03-18 — 08:34 AEST

**Symptom:** `cost_summary_30d.json` showed $0 advertising on 2026-03-19 through 2026-03-21, contradicting `ad_performance_7d.json` which showed $76.77 spend on 2026-03-20.
**Root cause:** Timing race — cost-collector cron ran at 01:00 AEST with `--days 2` (yesterday+today). The fb-metrics-collector runs at 12:00 and 23:00. If cost-collector runs before the day's ad metrics are written, it records $0 and never backfills because `--days 2` only covers yesterday and today.
**Fix:** (1) Ran `cost-collector.py --days 5` manually to backfill the gap. (2) Updated cron from `--days 2` to `--days 4` so it always re-collects the last 4 days, preventing stale $0 entries.
**Files:** crontab updated
**Recurrence:** First occurrence — cost-collector has been running since creation but the timing race was never caught.
