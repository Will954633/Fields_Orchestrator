## [EMAIL-DRY-RUN] Add local dry-run reply/send preview to email wrapper — 22:10 AEST
**Symptom:** The orchestrator email wrapper could pass through to the email CLI, but there was no local preview path for `reply`/`send` when `--dry-run` was requested from the VM wrapper.
**Root cause:** `scripts/fields-email.py` only special-cased `read` and `thread`; all other commands were delegated straight to `email_cli.py`, which meant the wrapper did not provide a draft-first preview layer.
**Fix:** Added wrapper-level dry-run handling for `reply` and `send` so the VM can preview the exact payload locally without invoking live send logic. Also updated the wrapper usage text and imported `argparse` for the new parsing path.
**Files:** `scripts/fields-email.py`
**Recurrence:** First
