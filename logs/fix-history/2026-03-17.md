## [CEO-AGENT-REFACTOR] CEO agent system refactor — 5 bugs fixed — 10:00 AEST

**Symptom:** CEO agent system had 5 structural issues identified during post-build review.

**Root cause / Fix (per issue):**
1. **Two launchers with diverging logic** — `ceo-agent-launcher.py` (npx/local, broken) and `ceo-agent-launcher-remote.sh` (SSH, working but buggy). Fixed by rewriting `.py` to use SSH to property-scraper (matching .sh's execution model) with all other bugs fixed. `.sh` deleted.
2. **One MongoDB connection per metric** — `ceo-context-export.py` spawned 6+ separate Python subprocesses for metrics. Fixed by consolidating all 8 metric/experiment/changes queries into one `query_mongodb()` call, replacing `export_experiments()` and `export_recent_changes()` with the unified `export_metrics()`.
3. **No upsert — duplicate proposals on rerun** — `insert_one` had no existence check. Fixed with `update_one({agent, date}, $set, upsert=True)` so reruns are idempotent.
4. **Context snapshot pushed to sandbox repo** — `push_proposals_to_github` never cleaned up the `context/` copy that `setup_workspace()` planted inside the sandbox dir. Fixed: new `push_to_github()` runs `rm -rf context` before `git add -A`.
5. **Per-file SHA lookup (2 API calls per file)** — `gh_api_put` did a `gh api GET` to fetch the SHA before every PUT. Fixed by adding `load_sha_cache()` which fetches all SHAs in one tree API call at run start; `gh_api_put` now reads from `_SHA_CACHE` dict.

**Files:** `scripts/ceo-agent-launcher.py` (rewritten), `scripts/ceo-context-export.py` (edited), `scripts/ceo-agent-launcher-remote.sh` (deleted), crontab updated.
**Recurrence:** First occurrence (post-build review).
