# Fix History — 2026-04-26 (AEST)

## [MIGRATION-CUTOVER-001] Cut Azure Cosmos → local MongoDB cutover for orchestrator services — 20:36 AEST
**Symptom:** Data was migrated from Azure Cosmos to local MongoDB on the VM, but the orchestrator daemon, trigger-poller, watchdog, valuation-api, and valuation-poller were all still connecting to Azure Cosmos. OPS_STATUS dashboard showed "Last run date: None" because the dashboard reads from local MongoDB but the daemon was still writing to Azure. Last successful pipeline run was 2026-04-22 (pre-migration); 3-day gap (Apr 23–25) followed.
**Root cause:** Three layers of stale Azure config overrode the new `.env` localhost URI:
1. `/etc/systemd/system/fields-orchestrator.service` had `Environment="COSMOS_CONNECTION_STRING=mongodb://...azure.com..."` hardcoded — systemd `Environment=` overrides values loaded from `.env` by python-dotenv.
2. `/etc/systemd/system/fields-claude-agent.service` had the same hardcoded Azure URI.
3. `config/settings.yaml` `mongodb.uri` still pointed at Azure Cosmos.
4. `src/orchestrator_daemon.py:107` had a defensive guard `raise ValueError("MongoDB URI must not be localhost in cloud deployment")` — this rejected the new localhost URI after the env was fixed.
**Fix:**
- Backed up the two systemd units to `*.bak.cosmos` and rewrote the `Environment="COSMOS_CONNECTION_STRING=..."` lines to point at `mongodb://fieldsadmin:***@localhost:27017/?authSource=admin`.
- Backed up `config/settings.yaml` to `settings.yaml.bak.cosmos` and updated `mongodb.uri` to the local URI.
- Removed the localhost-rejection guard in `src/orchestrator_daemon.py` (lines 100–107) — Azure has been cut off entirely, localhost is now the canonical destination.
- `systemctl daemon-reload` and restarted all 5 services. Confirmed via `/proc/$PID/environ` that the daemon now has `COSMOS_CONNECTION_STRING=mongodb://...@localhost:27017/...`.
**Files:**
- `/etc/systemd/system/fields-orchestrator.service` (URI changed)
- `/etc/systemd/system/fields-claude-agent.service` (URI changed)
- `config/settings.yaml` (URI changed; not pushed to GitHub per CLAUDE.md rule — contains credentials)
- `src/orchestrator_daemon.py` (removed cloud-deployment guard, lines 100–107)
