## [APPR-V4-LAYER2-PHASE2-01] Fit cascade — auto-compact variant on overflow (Layer 2 Phase 2) — 11:24 AEST

**Goal:** When fit-check detects a section overflowing, automatically apply a compact CSS variant to that section and re-check. Sections that fit at standard variant render at standard. Sections that overflow render at compact. No renderer changes, no re-render — pure CSS swap.

**What shipped:**

1. **`data-variant="standard"` attribute** added alongside `data-section` on every templated section's `<div class="page">`. CSS selector `.page[data-variant="compact"]` activates compact layout when the attribute flips.

2. **Compact CSS block in `preview.html`** — ~30 selectors covering the most layout-significant levers:
   - `.page-pad`: padding 18mm/20mm/14mm → 13mm/17mm/10mm
   - `.right-headline`: 28pt → 24pt; `.s03` variant: 26pt → 22pt
   - `.right-subhead`: smaller + tighter line-height
   - `.fields-advantage`: 3.5mm padding → 3mm; `.fa-body`: 9.5pt → 8.5pt
   - `.spread-number`: 84pt → 64pt
   - `.spread-headline`: 28pt → 26pt (and `.long`: 24pt)
   - `.cohort-anchor`, `.evidence-row`, `.synthesis`, `.persona-card`, `.match-bar-label`, `.willingness*` all tightened proportionally
   - Selectors use `!important` so they override the inline styles in the section templates

3. **Cascade orchestration in `generate_appraisal_v4.py`:**
   - Pass 1: render all sections standard, splice, write HTML, run fit_check.js
   - If any section reports `status: "overflow"`, swap its `data-variant="standard"` → `"compact"` in the HTML text
   - Pass 2: re-run fit_check on the modified HTML
   - Final result: audit JSON includes `fit_passes: [...]` (per-pass summary) and `variants_applied: {section_key: variant}` (which sections needed compact)
   - PDF renders from the final HTML

**Out of scope (Phase 2.5 / Phase 3 if needed):**
- `ultra_compact` variant fallback if `compact` still overflows

## [APPR-V4-LAYER2-PHASE1-01] Browser-based section overflow detector (Layer 2 Phase 1) — 11:15 AEST

**Goal:** Catch dynamic-content layout failures that pure-Python content budgets can't predict (font widths vary, wrapping is unpredictable, images can be intrinsic-size-dependent). Phase 1 of Layer 2 ships the measurement; Phase 2 will add the compact-variant fit cascade.

**What shipped:**

1. **`data-section` attributes** on every templated section's `<div class="page">` (8 explicit sections + recommendation_p11/p18 via Jinja interpolation). The cover page uses `<div class="cover">` and is excluded by design.

2. **`scripts/appraisal_template/fit_check.js` (new):** Node script using `puppeteer-core` + system Chrome. Loads the spliced HTML at A4 dimensions (794×1123px @ 96dpi) with `emulateMediaType('print')` for accurate sizing. For each `[data-section]` element:
   - Toggles `overflow:visible` and `.page-pad { height: auto }` briefly
   - Iterates direct children of `.page-pad` and measures `getBoundingClientRect().bottom`
   - Compares to `.page`'s `clientHeight` (1123px)
   - Classifies: `ok` (≤50px), `tight` (51-200px), `overflow` (>200px) — tolerance accounts for known browser/PDF render drift

3. **Generator integration:** `generate_appraisal_v4.py` calls `fit_check.js` between splice and PDF render. Captures the JSON report into the audit payload at `audit.fit_check`. CLI prints summary: `Fit-check: 10 pages measured · 0 overflow · 0 tight` plus per-page overflow detail when present.

4. **§01R satellite image constraint:** the fit-check immediately flagged §01R overflowing by ~275-295px in browser mode (the 1280×1280 fallback satellite image renders at 644px tall when `width:100%` and `height:auto`). Constrained to `max-height:88mm` + `object-fit:cover` so the image always fits in roughly 30% of the page regardless of intrinsic dimensions. After fix: 0 overflow on both 5 Pinnacle Court and 13TC.

**Audit JSON now includes:**
```json
{
  "fit_check": {
    "viewport": {"width": 794, "height": 1123},
    "sections": [
      {"section_key": "01_right", "page_index": 2, "client_height_px": 1123,
       "scroll_height_px": 1095, "overflow_px": 0, "status": "ok"},
      ...
    ],
    "summary": {"sections_measured": 10, "tight": 0, "overflows": 0}
  }
}
```

**Files:** `scripts/appraisal_template/fit_check.js` (new), `scripts/appraisal_template/render.py`, `scripts/generate_appraisal_v4.py`.
