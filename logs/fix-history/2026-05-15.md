## [APR01-MERGE-01] Non-destructive merge: apr01 snapshot → live Gold_Coast — 12:00–12:06 AEST
**Symptom:** Live `Gold_Coast.<suburb>` collections had lost `scraped_data` blocks (images, timeline, valuation, rental_estimate) for the off-market property cohort. Investigation today found Will's local Mongo still had this data intact for 214,840 properties across 81 suburbs (3.48M image URLs, 979K timeline entries, all on Domain CDN). Mongo dump from 2026-04-01 was SCP'd to VM.
**Root cause:** The Cosmos→Mongo migration (2026-04-12) appears not to have carried the full `scraped_data` block across for off-market records. Image URLs were never dead — they were lost from the live DB. The Azure Blob shutdown was a red herring for this dataset.
**Fix:** mongorestore --gzip --archive of the 261 MB Apr 1 dump into side namespaces (Gold_Coast_apr01, Gold_Coast_Recently_Sold_apr01, etc.) with --noIndexRestore for cross-version safety (snap was Mongo 8.0.4, VM is 7.0.31). Then bulk merge into live by `_id` — for every snap doc with `scraped_data.images.0` populated, wrote the full snap `scraped_data` to live `scraped_data_apr01_recovered` sidecar + `apr01_recovery_at` + `apr01_recovery_source: "mongodump_20260401"`. Idempotent (skips docs already recovered). Order: target suburbs first (Robina, Merrimac, Varsity Lakes, Burleigh Waters), then remaining 77.
**Results:** 214,840 scanned, **204,702 recovered**, 10,138 missing_live (all Surfers Paradise — see SURFERS-GAP-01 below), 0 errors. End-to-end verified: sample URL returns HTTP 200 image/webp 152KB from rimh2.domainstatic.com.au.
**Files:** ad-hoc Python merge script (not retained), log at `logs/apr01_merge_20260515_1200.log`, side namespaces in local Mongo (`Gold_Coast_apr01`, `Gold_Coast_Recently_Sold_apr01`, `Gold_Coast_Currently_For_Sale_apr01`, `Gold_Coast_URL_Tracking_apr01`, `Gold_Coast_POIs_apr01`, `property_data_apr01`), archive at `/tmp/mongodump_apr01.archive.gz` (274 MB).
