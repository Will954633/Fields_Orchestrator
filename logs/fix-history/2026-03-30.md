## [VOICE-IDLE-TIMEOUT] Auto-end listening after 30s of no speech — 09:22 AEST
**Symptom:** Voice agent stays in listening mode indefinitely if the user doesn't speak, keeping the mic open and the call active with no timeout.
**Root cause:** The VAD only triggers `sendAudio()` after detecting speech + silence. If no speech is ever detected (user walks away, leaves mic open), the `listening` phase never ends.
**Fix:** Added a 30-second idle timeout in `beginListening()`. If no speech is detected within 30s, the call auto-ends (stopVAD + stopRecorder + endCall). The timer is cancelled as soon as speech is detected, and cleared on manual endCall.
**Files:** `voice-agent/web/index.html`
**Recurrence:** First
