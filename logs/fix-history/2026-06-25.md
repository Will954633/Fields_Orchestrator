# Fix History — 2026-06-25 (AEST)

## [MINI-SITE-VAL-PHOTOS] Comparable photos on the mini-site Valuation tab — 11:58 AEST
**Symptom:** The `/your-home` Valuation tab listed each comparable sale the home is being compared to, but showed no photos of those homes — users couldn't see what their property was being measured against. The card had large unused space from centre to right.
**Root cause:** `valuation_evidence_from_engine()` emitted no photo data for comps. The engine only attaches Azure-blob image URLs to each comp (`fieldspropertyimages.blob.core.windows.net`), which return **403 publicly** (same root cause as [HOUSE-MINI-SITE-009] — Azure blob account disabled). The frontend `EvidenceComparable` type had no photo field.
**Fix:**
- **Backend** ([scripts/property_reports/slot_resolver.py](../../scripts/property_reports/slot_resolver.py)): `valuation_evidence_from_engine()` now attaches up to 15 publicly-displayable photos per comparable as `{thumb, full}` pairs. New helpers `_comp_source_doc()` (resolves the comp's Gold_Coast source doc by `_id` → address fallback, suburb parsed from the comp address since comps can be cross-suburb) and `_comp_display_photos()` lift the comp's **Domain CDN** URLs — the same source the for-sale gallery serves: small signed `rimh2` thumbnails for the card strip, full-res `bucket-api` images for the lightbox, paired by Domain image-id stem. The Azure-blob mirror is deliberately excluded.
- **Frontend** (`Website_Version_Feb_2026`): added `photos?: {thumb,full}[]` to `EvidenceComparable`; `ValuationEvidence.tsx` now renders a thumbnail strip in the card's centre-right spare space (up to 4 thumbs + a `+N` overlay) and opens the existing `PhotoLightbox` on click (full-screen, `1/15` counter, prev/next, keyboard + swipe, scrollable thumbnail strip). New `ValuationEvidence.module.css` (responsive: thumbnails stack full-width below the text on mobile).
- **Backfill:** attached photos to the 10 existing `system_monitor.property_reports` reports that carry engine evidence (80/80 comps resolved photos) — surgical update of `valuation.evidence.comparables`, no recomputation of figures.
- Restarted `fields-property-report-poller` so new resolutions pick up the resolver change.
**Verification:** Resolver output checked against a real Robina valuation (8 comps × 15 photos, URLs return 200). Frontend typecheck + production build clean. Local visual verification via a Vite dev render of the template slug (temporary fixture injection, reverted): confirmed desktop centre-right thumbnails, `+11` overlay, lightbox, and mobile stacking.
**Files:** scripts/property_reports/slot_resolver.py; (website) src/pages/YourHomePage/data/homeFixture.ts, src/pages/YourHomePage/components/ValuationEvidence.tsx, src/pages/YourHomePage/components/ValuationEvidence.module.css.
**Recurrence:** First.

## [MINI-SITE-VAL-MIDPOINT-LABEL] Label midpoint value above the dot on the Valuation range bar — 14:20 AEST
**Symptom:** On the `/your-home` Valuation tab the range bar labelled the lower and upper bounds above its ends (e.g. $1,400,000 / $1,800,000) but the midpoint dot on the bar had no printed value — the midpoint figure only appeared in the prose below.
**Root cause:** Not a bug — the bar was built with lo/hi labels and a midpoint dot only; no label element was rendered above the dot.
**Fix:** In `ValuationEvidence.tsx` added a bold midpoint label inside the range-bar container, absolutely positioned at the same `midPct%` (`left`, `transform: translateX(-50%)`) as the dot and raised above the bar (`top: -26`) so it lands on the same line as the lo/hi labels. Bold (`fontWeight: 700`), copper (`var(--fields-copper)`) to tie it to the copper dot, `whiteSpace: nowrap`. Reuses existing `fmt$(mid)` / `midPct` — no new state or layout reflow.
**Files:** (website) src/pages/YourHomePage/components/ValuationEvidence.tsx. Commit 0727c3c → `Will954633/Website_Version_Feb_2026`.
**Recurrence:** First.

## [MINI-SITE-VAL-PHOTOS-SELFHOST] Self-host comparable photos from our own blob store — 12:35 AEST
**Symptom:** Comparable photos on the Valuation tab were served from Domain's CDN (`bucket-api.domain.com.au` / `rimh2.domainstatic.com.au`). Will asked why, given we pulled these photos locally.
**Root cause:** Our local blob store (`/data/blobs`, served at `blobs.fieldsestate.com.au`) only kept **150px cadastral thumbnails** for sold homes — full-res was retained only for currently-listed properties (`for_sale/`). So the lightbox needed Domain's full-res, and the whole site already served property photos from Domain CDN after the Azure blob account was disabled (see [IMAGE-OUTAGE-001]). Domain URLs can rotate/404 over time.
**Fix:**
- New `scripts/backfill_sold_fullres_photos.py`: for sold homes in core suburbs, downloads up to 15 full-res images from `property_images_original` (Domain bucket-api → b.domainstatic) and writes BOTH `property-images/sold/<suburb>/<id>/photos/<NN>.jpg` (full-res) and `.../thumbs/<NN>.jpg` (~320px JPEG, generated with PIL) to the local blob store — keyed by Mongo `_id` (== the engine comp `id`). Idempotent; concurrent; flags `sold_fullres_localized` on the doc.
- `slot_resolver.py`: new `_comp_local_photos()` builds `blobs.fieldsestate.com.au` `{thumb,full}` pairs (thumb from `thumbs/`, full from `photos/`) when a home has a localised gallery with both sizes; `valuation_evidence_from_engine()` now prefers it and falls back to the Domain CDN path (`_comp_display_photos`) only for homes not yet localised. Active comps (no `thumbs/`) stay on Domain for now.
- **Phase 1:** localised the 44 distinct sold comps backing the 10 live reports; re-backfilled `system_monitor.property_reports` so 75/80 comps now serve from our blob store (remainder are active listings → Domain).
- **Phase 2 (running):** full core-suburb backfill (robina, burleigh_waters, varsity_lakes, merrimac — ~1,390 sold homes) so future reports are self-hosted.
**Verification:** Live `/your-home/41-olympus-drive-robina` Valuation tab now serves all 8 comps from `blobs.fieldsestate.com.au` (full 805KB jpeg, thumb 11KB jpeg, both 200). Screenshots confirm thumbnails + full-res lightbox render from our own host. Thumbnails 320×222 (~18KB), full-res 1556×1080+ (~0.6MB).
**Files:** scripts/backfill_sold_fullres_photos.py (new); scripts/property_reports/slot_resolver.py.
**Recurrence:** First. Builds on [MINI-SITE-VAL-PHOTOS] (same day).
