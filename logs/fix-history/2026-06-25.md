# Fix History — 2026-06-25 (AEST)

## [MINI-SITE-VAL-PHOTOS] Comparable photos on the mini-site Valuation tab — 11:58 AEST
**Symptom:** The `/your-home` Valuation tab listed each comparable sale the home is being compared to, but showed no photos of those homes — users couldn't see what their property was being measured against. The card had large unused space from centre to right.
**Root cause:** `valuation_evidence_from_engine()` emitted no photo data for comps. The engine only attaches Azure-blob image URLs to each comp (`fieldspropertyimages.blob.core.windows.net`), which return **403 publicly** (same root cause as [HOUSE-MINI-SITE-009] — Azure blob account disabled). The frontend `EvidenceComparable` type had no photo field.
**Fix:**
- **Backend** ([scripts/property_reports/slot_resolver.py](../../scripts/property_reports/slot_resolver.py)): `valuation_evidence_from_engine()` now attaches up to 15 publicly-displayable photos per comparable as `{thumb, full}` pairs. New helpers `_comp_source_doc()` (resolves the comp's Gold_Coast source doc by `_id` → address fallback, suburb parsed from the comp address since comps can be cross-suburb) and `_comp_display_photos()` lift the comp's **Domain CDN** URLs — the same source the for-sale gallery serves: small signed `rimh2` thumbnails for the card strip, full-res `bucket-api` images for the lightbox, paired by Domain image-id stem. The Azure-blob mirror is deliberately excluded.
- **Frontend** (`Website_Version_Feb_2026`): added `photos?: {thumb,full}[]` to `EvidenceComparable`; `ValuationEvidence.tsx` now renders a thumbnail strip in the card's centre-right spare space (up to 4 thumbs + a `+N` overlay) and opens the existing `PhotoLightbox` on click (full-screen, `1/15` counter, prev/next, keyboard + swipe, scrollable thumbnail strip). New `ValuationEvidence.module.css` (responsive: thumbnails stack full-width below the text on mobile).
- **Backfill:** attached photos to the 10 existing `system_monitor.property_reports` reports that carry engine evidence (80/80 comps resolved photos) — surgical update of `valuation.evidence.comparables`, no recomputation of figures.
- Restarted `fields-property-report-poller` so new resolutions pick up the resolver change.
**Verification:** Resolver output checked against a real Robina valuation (8 comps × 15 photos, URLs return 200). Frontend typecheck + production build clean. Local visual verification via a Vite dev render of the template slug (temporary fixture injection, reverted): confirmed desktop centre-right thumbnails, `+11` overlay, lightbox, and mobile stacking.
