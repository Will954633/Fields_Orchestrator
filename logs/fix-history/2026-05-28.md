# Fix History — 2026-05-28

## [BLOB-BACKUP] GCS backup for /data/blobs — 14:35 AEST
**Symptom:** Property images (249GB, ~994k files) live only on VM disk at `/data/blobs/`. Azure Blob (previous off-VM copy) is disabled and being deleted. Single point of failure.
**Root cause:** Migration from Azure removed the redundant copy without setting up a replacement.
**Fix:**
- Created GCS bucket `gs://fields-blob-backup` (Nearline, australia-southeast1, uniform access).
- Kicked off initial full sync via `gsutil -m rsync -r /data/blobs/ gs://fields-blob-backup/` (background, logs at `/var/log/blob-backup/initial-sync.log`).
- Installed daily cron at 03:00 AEST for incremental `rsync -d` (mirror deletes too).
**Files:** crontab (added one line), `/var/log/blob-backup/` (new log dir).
**Recurrence:** First.
**Cost:** ~$3/month Nearline storage at 300GB.
**Next:** Verify initial sync completes (a few hours). Then Azure subscription can be cancelled.
