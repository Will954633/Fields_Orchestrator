## [TRACK-502] Appraisal HTML reports 502 in ops view — 09:35 AEST
**Symptom:** `https://vm.fieldsestate.com.au/track/view/<id>` returned `502 Bad Gateway (nginx)`. All `/track/*` endpoints (viewer, pixel, status, logo, page) were down — appraisal HTML reports would not render.
**Root cause:** nginx had no `location /track/` block, so `/track/...` fell through to `location /` and was proxied to the ttyd terminal (`https://127.0.0.1:7681`), which failed the upstream SSL handshake → 502. The tracking server itself (Flask, `127.0.0.1:3051`) was healthy the whole time. The `/track/` proxy block was lost when `sites-enabled/fields-vm` was rewritten on 2026-05-26 (audio-library edit). Note: `sites-enabled/fields-vm` is a standalone regular file, NOT a symlink to `sites-available/` — editing sites-available has no effect.
**Fix:** Added `location /track/ { proxy_pass http://127.0.0.1:3051/; ... }` to `/etc/nginx/sites-enabled/fields-vm` (trailing slash strips the `/track` prefix → server.py routes `/view`, `/page`, `/status`, etc.). Reloaded nginx. Also moved a stray `fields-vm.bak.*` out of `sites-enabled/` (it was being loaded via `include sites-enabled/*` and caused a conflicting-server-name warning) into `/etc/nginx/backups/`. Verified: `/track/view/<id>` → 200 (renders "27 Huntingdale Crescent, Robina"), `/track/page/<id>/1` → 200 image/png, `/track/health` → 200.
**Files:** /etc/nginx/sites-enabled/fields-vm
