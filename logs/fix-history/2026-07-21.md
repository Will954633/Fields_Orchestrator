## [OFFMARKET-DIRECT-TEST-VALUE-BUG] findOneAndUpdate .value misread silently burned all 5 direct-test admission slots, including one real visitor — 13:20 AEST
**Symptom:** Built a new 5-user experiment (`off-market-direct-test.mjs` + `off-market.$slug.tsx` loader) that should route a capped sample of real organic-Google visitors to `/off-market/:slug` straight into the free mini-site build instead of the $15 Square gate. First live verification requests all returned the normal $15 teaser page (no redirect), with no thrown error.
**Root cause:** `mongodb` v7 (this repo's driver version) changed `findOneAndUpdate()` to return the matched document directly (or `null`) — the legacy `{ value: doc }` wrapper is gone since driver v6. The code checked `result?.value`, which is always `undefined` even on a successful match, so every admission attempt read as "not admitted" and returned early — **but the atomic `$inc`/`$push` write had already been applied** by the same `findOneAndUpdate` call before that check ran. Every attempt (4 of mine debugging it, plus one genuine real visitor who clicked a real Google result to "5 Palmyra Court, Burleigh Waters" at 12:40pm AEST) silently consumed a real slot in `system_monitor.offmarket_direct_test_counters` with zero completed admissions — no mini-site built, no redirect served, no Telegram sent to Will. The real visitor saw the ordinary $15 teaser and was never routed into the test or logged anywhere visible.
**Fix:** Removed the `.value` access — `result` IS the document (or null). Also added a self-healing bootstrap (`$setOnInsert` upsert + retry) so the counter doc no longer needs a manual seed step before first use. Archived the 5 wasted (bug-consumed, never-completed) admission records to `system_monitor.offmarket_direct_test_counters_archive` and reset the live counter to `count: 0, admissions: []` so the real 5-slot experiment starts clean. Verified end-to-end with one real curl-simulated admission (302 to `/your-home/:slug`, `property_reports.direct_test` block written, `leads` row written, counter incremented correctly) — then deleted that verification's test artifacts and reset the counter back to 0 so it doesn't count against Will's real 5.
**Files:** `netlify/functions/off-market-direct-test.mjs`, `src/routes/off-market.$slug.tsx` (Website_Version_Feb_2026 repo)
**Recurrence:** First (new code, driver-version behavior change caught same-session before any real user impact accumulated beyond the single missed visitor above).

## [OFFMARKET-DIRECT-TEST-REFERER-OVERCORRECTION] Bot fix above wrongly rejected a real visitor within minutes — 14:40 AEST
**Symptom:** Will reported a real visitor to `/off-market/120-christine-avenue-burleigh-waters` got the old $15-teaser page, not the direct-test flow.
**Root cause:** The bot fix immediately above (OFFMARKET-DIRECT-TEST-BOT-ADMISSION) added a requirement that the referer contain a search query (`?q=`) or a `google.*/url?` redirect wrapper, reasoning that real click-throughs always carry one. A bare-origin Google referer is now the NORMAL shape for real organic search click-throughs, not a bot signature — the previous fix conflated "bare referer" (common, benign) with the actual bot tell from the 37 Roundelay Drive case (AWS-datacenter IP + zero client JS ever running), which are unrelated signals.
**Fix:** Removed the referer-shape requirement entirely. Kept the User-Agent bot-pattern check (still the correct signal — real browsers always send a `Mozilla` token and a recognizable browser fingerprint) and the IP exclusion list. Domain-match on `google.` is the only referer check now.
