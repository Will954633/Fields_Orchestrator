## [OCCUPANCY-FOR-SALE-BLIND-SPOT] Occupancy classifier + Telegram alert ignored active for-sale listing — 08:02 AEST
**Symptom:** Will got a Telegram "New Analyse Your Home lead" alert for 4/44 Frascott Avenue, Varsity Lakes that said "purchased 2020, no rental listing since" but gave no indication the property is currently listed for sale (Expressions of Interest, `listing_status: for_sale` in `Gold_Coast.varsity_lakes`).
**Root cause:** `occupancy_classifier.refresh_and_classify()` already fetches the matching Gold_Coast doc via `_find_gc_doc()` (which carries `listing_status`), but only used it to build the Domain profile URL for the Bright Data timeline fetch. The occupancy verdict was derived purely from the sale/rental timeline, so `listing_status` was never surfaced into `occ["signals"]` or the Telegram message.
**Fix:** In `refresh_and_classify()`, added `occ["currently_for_sale"] = gc_doc.get("listing_status") == "for_sale"` and appended an explicit signal string when true. In `build_property_report._notify_new_lead()`, added a `🏷️ Currently listed for sale` line to the Telegram alert whenever `occ.get("currently_for_sale")` is set.
**Files:** `scripts/property_reports/occupancy_classifier.py`, `scripts/property_reports/build_property_report.py`
**Recurrence:** First
