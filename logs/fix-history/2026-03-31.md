## [CEO-LAUNCHER-IMPORT] shared module not found in ceo-agent-launcher.py — 08:58 AEST
**Symptom:** Chat Agent queue and task processing failed with `No module named 'shared'` after agent runs complete.
**Root cause:** `scripts/ceo-agent-launcher.py` runs from the `scripts/` directory (Python sets `sys.path[0]` to the script's own directory). The `shared` package lives at the project root (`/home/fields/Fields_Orchestrator/shared/`), which was never added to `sys.path`. Top-level imports (`ceo_agent_lib`, `validate_snapshot`) worked because they're in `scripts/`, but the deferred `from shared.db import get_client` calls inside `_check_and_send_telegram`, `_process_deploy_manifests`, and `_process_will_tasks` all failed.
**Fix:** Added `sys.path.insert(0, _PROJECT_ROOT)` near the top of the script (after `Path` is imported, before any imports that might need it). This ensures `shared.*` is importable regardless of how the script is invoked.
**Files:** `scripts/ceo-agent-launcher.py`
**Recurrence:** First

## [CEO-LAUNCHER-PUSH] GitHub sandbox push uses git push instead of gh api — 08:58 AEST
**Symptom:** CEO agent launcher failed to push deliverables to `Will954633/fields-ceo-sandbox` with `remote: Invalid username or token. Password authentication is not supported for Git operations.`
**Root cause:** `push_to_github()` in `ceo-agent-launcher.py` ran `git push origin main` on the property-scraper VM, which has no valid git credentials for HTTPS push. All GitHub operations on these VMs must use `gh api` (Contents API).
**Fix:** Replaced `git add -A && git commit && git push` with file-by-file upload via `gh api repos/.../contents/PATH --method PUT`. Uses a helper Python script on the remote VM to handle base64 encoding and JSON payload construction for large files.
**Files:** `scripts/ceo-agent-launcher.py`
**Recurrence:** First
