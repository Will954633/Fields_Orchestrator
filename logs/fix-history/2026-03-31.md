## [CEO-LAUNCHER-IMPORT] shared module not found in ceo-agent-launcher.py — 08:58 AEST
**Symptom:** Chat Agent queue and task processing failed with `No module named 'shared'` after agent runs complete.
**Root cause:** `scripts/ceo-agent-launcher.py` runs from the `scripts/` directory (Python sets `sys.path[0]` to the script's own directory). The `shared` package lives at the project root (`/home/fields/Fields_Orchestrator/shared/`), which was never added to `sys.path`. Top-level imports (`ceo_agent_lib`, `validate_snapshot`) worked because they're in `scripts/`, but the deferred `from shared.db import get_client` calls inside `_check_and_send_telegram`, `_process_deploy_manifests`, and `_process_will_tasks` all failed.
**Fix:** Added `sys.path.insert(0, _PROJECT_ROOT)` near the top of the script (after `Path` is imported, before any imports that might need it). This ensures `shared.*` is importable regardless of how the script is invoked.
**Files:** `scripts/ceo-agent-launcher.py`

## [AGENT-APPROVAL-BROKEN] Chat Agent cannot display or process agent approval requests — 13:56 AEST
**Symptom:** When CEO agents sent decision-requesting messages, Chat Agent showed them as plain text with no approve/deny buttons. Typing "approve" in chat also failed — router had no context about which message to approve.
**Root cause:** Three interconnected bugs: (1) Agent messages requesting decisions were sent as `mid_session_message` type, but UI only showed approve/deny buttons for `deploy_approval` type. (2) Viewing any agent message immediately marked it `delivered`, which removed it from the poller's pending cache — so the router lost all context about agent messages the moment they were displayed. (3) `respond_to_approval()` rejected any message not in `pending`/`pending_approval` status, but messages were already `delivered` by the time user tried to approve.
**Fix:** Four changes: (1) `agent_poller.py` — added `_session_messages` cache that retains delivered messages for the session; new `get_session_messages()` returns pending + session messages; `respond_to_approval()` now accepts `delivered` status; bridge trigger works for `mid_session_message` type too. (2) `index.html` — UI detects decision-requesting messages (containing QUESTION+OPTIONS, APPROVE, APPROVAL NEEDED keywords) and shows approval cards with buttons. (3) `router.py` — `_format_agent_messages()` labels decision-requesting messages as "DECISION REQUESTED" with full content (500 chars vs 300). (4) `server.py` — router context and API endpoint now use `get_session_messages()` instead of `get_pending_messages()`; fixed datetime serialization bug in `/api/todos` endpoint.
**Files:** `voice-agent/agent_poller.py`, `voice-agent/server.py`, `voice-agent/router.py`, `voice-agent/web/index.html`
**Recurrence:** First
