# Fix History — 2026-06-17

## [GDRIVE-MCP-FORMAT] gdrive MCP discarded HTML/markdown formatting on Google Docs — 10:45 AEST
**Symptom:** Creating/updating a Google Doc via the gdrive MCP (`create_file`/`update_file`) saved the raw source as literal text — `**bold**`, `<h1>` tags, `| table |` pipes, and `---` all appeared verbatim in the document instead of rendering as headings/bold/tables. Hit while generating Will's real-estate resume into the `Will_Simpson` Drive folder.
**Root cause:** Both handlers in `mcp-servers/gdrive/index.mjs` hard-coded the upload media mimeType to `text/plain` whenever the target was a Google Workspace doc (`isGoogleWorkspace(...) ? "text/plain" : ...`). Drive only converts uploaded media into a formatted Doc when the media mimeType is a convertible type (`text/html` / `text/markdown`); forcing `text/plain` means Drive inserts the literal characters and never converts. `update_file` even accepted a `mimeType` param but ignored it for Docs.
**Fix:** Added `CONVERTIBLE_SOURCE_MIMES = ["text/html","text/markdown"]`. `update_file` now uploads with the caller's `mimeType` when it's convertible (else `text/plain`). `create_file` gained an optional `sourceMimeType` param so a Google Doc target can be created from HTML/markdown in one call. Both default to `text/plain` so existing callers are unaffected. Verified end-to-end: pushed `text/html` to the resume Doc and the export confirmed real H1/H2/H3 headings, a 2-col table, bold, italics, and bullet lists. `node --check` passes. (Resume itself was delivered via a one-off direct Drive API call because the running MCP process still had the old code; the fixed server takes effect on next MCP restart.)
**Files:** mcp-servers/gdrive/index.mjs
**Recurrence:** First (distinct from prior [GDRIVE-MCP] server-setup entry)
