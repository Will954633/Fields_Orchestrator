# Spread Template — V3 internal pages

Every internal spread is **two pages, one idea, one hero visual**. No "continued" pages. No third spread page. If a thought needs three pages, it isn't ready.

The unit of work is the hero visual on the right page. If the image, map, or diagram doesn't land in two seconds, the page doesn't ship.

---

```
│                                                │

│                                                │
│                                                │
│                                                │
│                                                │
│                                                │
│  Footer: F mark · "Smarter with data" · page #  │
└────────────────────────────────────────────────┘
```

### Editorial rules — left page

- **Headline must be a declarative sentence.** Never a question. Never hedged.
- **Body is one paragraph.** No bullets. No callout boxes. No evidence panel. The page is doing one job: stating a principle clearly.
- **No "What Fields uniquely does" list.** That list was the V2 mistake. The right page proves it; the left page only states it.
- **At least 50% of the page is breathing room.** Generous top margin. Generous left margin. Headline sits at roughly the optical centre.
- **No more than three pieces of text on the page** — number, headline, paragraph (and optionally one supporting line). If it has four, cut one.

## Right page — *Applied to your home*

```
┌────────────────────────────────────────────────┐
│                                                │
│  Header strip                                  │
│   ─ FOR 13 TERRACE COURT (small, all-caps)     │
│   ─ Fields F mark (top-right)                  │
│                                                │
│                                                │
│  Headline finding                              │
│   ─ Short. Specific. Numerical when possible.  │
│   ─ 36–48pt cap-serif                          │
│                                                │
│                                                │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │                                          │  │
│  │                                          │  │
│  │           HERO VISUAL                    │  │
│  │           ─ image, map, chart, diagram   │  │
│  │           ─ full-width                   │  │
│  │           ─ ~50% of page height          │  │
│  │           ─ no caption clutter           │  │
│  │                                          │  │
│  │                                          │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  Source line (8pt, single line, gutter)        │
│                                                │
│                                                │
│  Supporting facts (5–7 lines maximum)          │
│   ─ Each line is short. Each carries one fact. │
│   ─ Format: one-em dash + tight phrase         │
│                                                │
│                                                │
│  Closing line (one sentence)                   │
│   ─ Emotional, not operational                 │
│   ─ Never references the next spread           │
│                                                │
│  Footer: F mark · "Smarter with data" · page #  │
└────────────────────────────────────────────────┘
```

### Editorial rules — right page

- **The hero visual is the page.** If the image isn't strong enough to carry the spread, redesign or reshoot — don't compensate with text.
- **Headline finding is one sentence, ideally with one number.** "1 of 4." "Top 22%." "Three competitors." Numbers become the headline.
- **Supporting facts are short lines, never sentences.** No more than seven. Each starts with an em-dash and a tight phrase.
- **No "what this means for your sale" paragraph.** V2 ended every right page with operational language. V3 ends with one emotional sentence.
- **No cross-references to other spreads.** Each spread closes on itself.
- **Source line is small and final.** One line. Below the visual. Never inside it.

---

## Visual system (carry from V2)

- **Cap-serif headlines** — same V2 face
- **Warm-grey body type** on **sand/cream** ground (never pure white)
- **Copper accent** — spread numbers, key statistics, footer F mark
- **Forest green panels** — used sparingly, only for one quote-style block on the closing page (used four times in V2; reduced to one use in V3)
- **No split-rule down the gutter.** V2 used this; V3 removes it. The two pages should feel continuous, not adversarial.
- **No "Only Fields" badge.** V2 introduced this; V3 removes it. The work proves itself.

### Typography targets (V3)

| Element | Size | Weight | Leading |
|---|---|---|---|
| Spread number | 96pt | hairline | — |
| Left headline | 56–72pt | regular cap-serif | 1.05 |
| Right headline | 36–48pt | regular cap-serif | 1.1 |
| Body paragraph | 13pt | regular | 1.7 |
| Supporting line | 11pt | italic OR small caps | 1.4 |
| Source line | 8pt | regular | 1.3 |

V2 body was effectively 10pt. V3 lifts to 13pt. This single change does more for "luxury" perception than any image swap.

---

## Word counts (target — V3)

| Element | Target |
|---|---|
| Left-page body paragraph | 50–80 words |
| Left-page supporting line | 5–25 words |
| Right-page headline finding | 6–14 words |
| Right-page supporting facts (total) | 30–60 words |
| Right-page closing line | 12–25 words |
| **Total per spread** | **120–200 words** |

V2 spreads (including the "continued" page) ran 600–900 words each. V3 spreads run 120–200. That **is** the redesign.

A V3 spread reads end-to-end in **30 seconds**. A skim takes **8 seconds** and pulls every headline.

---

## What is deliberately removed from the V3 anatomy

| V2 element | Why removed |
|---|---|
| "WHAT FIELDS UNIQUELY DOES" bullet panel | Telling, not showing. Right page proves it. |
| "THE EVIDENCE" callout | Distrustful. Cite under the visual instead. |
| Pull quote on every page | Loses meaning at frequency. Reserved for closing page. |
| "What this means for your sale" paragraph | Reads operational. Closing line carries the emotion. |
| Cross-reference to next spread | Stitches a deck. V3 wants standalone arguments. |
| "Continued" third page | The structural source of V2's density. |
| Split rule down gutter | Visual noise. Pages feel continuous in V3. |
| "Only Fields" copper badge | Self-congratulatory. Cut. |
