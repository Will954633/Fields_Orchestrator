# Spread Template — V2 internal pages

Every internal spread of the Property Position Report (Version Two) is built from the same two-page skeleton. The left page makes a general argument about a Fields capability; the right page applies that capability to the subject property with specific, defensible findings.

**Why fixed structure:** the seller learns the rhythm by spread 2 and reads ten times faster from spread 3 onward. The variation is the *content*, not the *form*.

---

## Left page — *The capability*

### Anatomy

```
┌────────────────────────────────────────────────┐
│  Header strip:                                 │
│   ─ Property address (small, top-left)         │
│   ─ Fields logomark (top-right)                │
│                                                │
│  Spread number                                 │
│   "01"  (large, copper)                        │
│                                                │
│  Headline                                      │
│   ─ Single declarative sentence                │
│   ─ 60-80pt serif                              │
│                                                │
│  Lede paragraph                                │
│   ─ 2-3 sentences. States the principle.       │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │  WHAT FIELDS UNIQUELY DOES               │  │
│  │   ─ 3 to 6 bullet items                  │  │
│  │   ─ Each bullet bolded lead + one sentence│  │
│  │   ─ Specifics, not vibes                 │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │  THE EVIDENCE                            │  │
│  │   ─ Counts, citations, named studies     │  │
│  │   ─ Hyperlinks where relevant            │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  Pull quote                                    │
│   ─ Will or external authority                 │
│   ─ Italic, copper rule above                  │
│                                                │
│  Why this matters for you                      │
│   ─ One paragraph closing the argument         │
│                                                │
│  Footer: "Smarter with data" + page number     │
└────────────────────────────────────────────────┘
```

### Editorial rules

- **Headline must be declarative** — never a question, never a hedge. "We see your home in more dimensions" not "How we see your home".
- **Bullets carry weight** — each one starts with a bold short noun phrase, then one sentence of plain English. No corporate adjectives.
- **Evidence panel must cite** — counts come with sources, studies come with year + author + N. If a number cannot be cited, it does not appear.
- **Pull quote either is from Will or names a published study** — never anonymous. Italic, single-line copper rule above and below.
- **Closing paragraph never sells** — it explains the *consequence* of the absence of this capability for an ordinary listing. The reader is left to draw the comparison.

## Right page — *Applied to your home*

### Anatomy

```
┌────────────────────────────────────────────────┐
│  Header strip:                                 │
│   ─ Property address (small, top-left)         │
│   ─ Fields logomark (top-right)                │
│                                                │
│  Section label                                 │
│   "FOR 13 TERRACE COURT"  (small, all-caps)    │
│                                                │
│  Headline finding                              │
│   ─ The one-sentence answer for this home      │
│   ─ Specific. Numerical. Suburb-named.         │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │  THE INVENTORY                           │  │
│  │   ─ Specific data points found           │  │
│  │   ─ Format depends on capability:        │  │
│  │      tabular for valuation               │  │
│  │      list for findings                   │  │
│  │      ranked panels for personas          │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  Visual                                        │
│   ─ Chart, map, table, image grid              │
│   ─ Always labelled with source + date         │
│   ─ Caption directly below in 9pt              │
│                                                │
│  What this means for your sale                 │
│   ─ One paragraph                              │
│   ─ States the lever this finding pulls        │
│   ─ References the spread that develops it next│
│                                                │
│  Footer: "Smarter with data" + page number     │
└────────────────────────────────────────────────┘
```

### Editorial rules

- **Address strip identical on every right page** — "FOR 13 TERRACE COURT" or equivalent.
- **Numbers always specific** — "658 m²", not "around 650". "0.34 internal-to-land ratio", not "high".
- **Visual must be labelled and dated** — every chart, map, and table carries its data source and date. No floating visuals.
- **Closing paragraph carries the cross-reference** — every right page ends by naming the next spread it connects to. This is what stitches the ten spreads into a single argument rather than a deck of slides.

## Visual system

Inherited from V1 (Marty designer style):
- **Copper accent** (used for spread number, callouts, key quotes)
- **Forest green dark panels** (used for evidence and key findings)
- **Sand/cream background** (warm, paperlike — *not* white)
- **Warm-grey body type** (forest green on cream, never pure black)
- **Header rule** is two-toned — copper underline beneath address strip on every spread

Two new visual signatures introduced for V2:
- **The split rule** — a thin gold rule down the gutter, signalling "left page argues, right page proves"
- **The "only Fields" badge** — a small copper roundel placed wherever a capability is genuinely unique to Fields (no competitor delivers it). Used sparingly to retain credibility.

## Word counts (target)

| Element | Target |
|---|---|
| Lede paragraph | 30-50 words |
| "What Fields uniquely does" bullets | 15-30 words each |
| Evidence panel | 40-60 words total |
| Pull quote | 20-40 words |
| Why this matters | 50-70 words |
| Right-page headline | 12-25 words |
| Inventory list | 40-80 words total |
| What this means for your sale | 50-80 words |

A spread should read end-to-end in roughly **90 seconds** of close reading. A skim should pull every headline, bullet bold, and pull quote in **15 seconds**.
