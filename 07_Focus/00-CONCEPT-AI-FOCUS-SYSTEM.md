# AI Focus System — Concept Exploration

> **Purpose:** Explore how AI can solve the solo-founder focus problem — knowing what to work on, when, and actually doing it. Not a development plan yet. Options, trade-offs, and honest analysis.
>
> **Date:** 2026-03-30
> **Author:** Will Simpson + Claude (Opus 4.6)
> **Status:** Exploration / Pre-development

---

## 1. The Problem (Stated Plainly)

Will identified four specific failure modes in how he works:

| # | Failure Mode | Pattern |
|---|-------------|---------|
| A | **Don't know what to work on when** | No system tells him "this is the highest-leverage thing right now" |
| B | **Can't see the critical path** | Milestones exist vaguely but the dependency chain — what must happen first — isn't mapped |
| C | **Avoids grind work** | Accounting, emails, admin — essential but boring. Gets deferred indefinitely |
| D | **Chases novelty over execution** | New concepts are exciting. Finishing existing work is not. The prototype graveyard grows. |

These aren't character flaws — they're information problems and incentive misalignments. Every solo founder has them. The difference is that Fields already has AI infrastructure that most businesses don't, which means we can actually build a system that addresses all four.

### The Deeper Insight

Will also identified something more universal: **if humans had perfect knowledge of what they should be doing, they'd almost certainly be doing something different right now.** This is the core design problem. The system we build isn't a to-do list with reminders. It's a decision engine that continuously answers: *given everything we know about the business, what is the single most valuable thing Will could do in the next 2 hours?*

## 2. What We Already Have (Capability Audit)

Before designing anything new, here's what exists:

### Systems That Generate Intelligence

| System | What It Produces | Frequency | Gap |
|--------|-----------------|-----------|-----|
| **CEO Agents** (5 agents) | Prioritised proposals with scores, dependencies, time horizons | Daily 02:00 AEST | Proposals aren't tracked for approval or outcome. No feedback loop. |
| **OPS_STATUS.md** | Pipeline health, coverage, experiment results, service status | Every 15 min | Reactive — tells you what's broken, not what to build |
| **Knowledge Base** | 1,644 searchable docs (strategy, meetings, books, code) | On-demand | Keyword search only. No "what's relevant to my current goal?" |
| **Ad Metrics** | Facebook + Google performance, spend, conversions | 2x/day | Collected but not synthesised into decisions |
| **Fix History** | Every bug fix with root cause, recurrence tracking | Per-fix | Good for pattern detection, not for prioritisation |

### Systems That Can Execute

| System | What It Does | Capacity | Gap |
|--------|-------------|----------|-----|
| **Fields Chat Agent** | 4 modes: quick answers, deep thinking, email ops, background tasks | 3 concurrent workers, 20-min timeout | No priority queue. Workers don't know what matters. |
| **Todo Manager** | CRUD tasks with priority, due dates, tags, Telegram reminders | Unlimited | Static priorities. No link to proposals or business goals. |
| **Trigger Poller** | Executes predefined pipeline processes on demand | Sequential | Only pipeline steps, not arbitrary work |
| **Email Agent** | Read, search, draft, reply, send | Via chat agent | No auto-triage, no follow-up tracking |
| **Orchestrator Pipeline** | 30-step nightly data processing | Scheduled 20:30 AEST | Reliable but not relevant to focus problem |

### What's Missing (The Gaps)

1. **No unified priority view.** Todos, proposals, tasks, and emails live in separate systems. There's no single answer to "what should I do next?"
2. **No goal → milestone → task decomposition.** Business objectives aren't broken into ordered steps.
3. **No approval workflow.** CEO agents propose, but there's no approve → execute → measure loop.
4. **No grind-work automation.** Email triage, accounting reconciliation, and admin tasks require Will to initiate them manually.
5. **No novelty detection.** Nothing flags when Will is drifting from the plan into a new tangent.
6. **No outcome tracking.** Completed work isn't measured against the goal it served.

## 3. What This System Could Be (Three Options)

### Option A: Focus Dashboard (Lightest Touch)

**Concept:** A single daily view — generated by the Chief of Staff agent — that synthesises all signals into "here are your 3 things today." Displayed in the chat agent and/or ops dashboard.

**How it works:**
- Chief of Staff agent (already exists) produces a daily brief
- Extend it to: (1) rank CEO proposals against stated milestones, (2) surface overdue grind tasks, (3) flag novelty drift if recent work doesn't map to any milestone
- Chat agent loads this as context in every conversation
- Will sees it first thing, checks off items, system tracks completion

**What it solves:**
- A (what to work on): Yes — ranked daily priorities
- B (critical path): Partially — proposals have dependencies, but no formal milestone map
- C (grind work): Partially — surfaces overdue admin, but doesn't do it for you
- D (novelty drift): Partially — flags drift, but doesn't prevent it

**Effort:** ~2-3 days. Extend existing Chief of Staff output + add a focus display to chat agent.

**Limitation:** This is still advisory. It tells Will what to do but can't make him do it. And without a milestone map, "priority" is just agent opinion, not path-to-goal logic.

---

### Option B: AI Chief of Staff (Medium Build)

**Concept:** An always-on AI agent that acts as Will's chief of staff — not just advising what to do, but managing the workflow. It tracks goals, decomposes them into milestones, sequences tasks, handles grind work autonomously, and flags when Will drifts.

**How it works:**

```
[Business Goals]  →  [Milestones]  →  [Weekly Sprints]  →  [Daily Focus Queue]
       ↑                                                          ↓
  [Quarterly Review]                                    [Execution + Tracking]
       ↑                                                          ↓
  [Outcome Data]  ←  [Completion + Impact Measurement]  ←  [Done Items]
```

**Components:**

1. **Goal Registry** — Will defines 3-5 business goals with target dates. Example:
   - "First paying customer by June 2026"
   - "100 unique weekly visitors from organic search by May 2026"
   - "All 3 target suburbs at 95%+ listing coverage by April 15"

2. **Milestone Decomposition** — AI breaks each goal into ordered milestones with dependencies. Example for "First paying customer":
   - M1: Valuation accuracy within 5% of sale price (backtest) ← *data quality*
   - M2: Pre-sale report template designed and tested ← *product*
   - M3: 3 agents contacted with sample reports ← *growth*
   - M4: Pricing model finalised ← *business*
   - M5: First sale closed ← *M1-M4 complete*

3. **Weekly Sprint Generator** — Each Monday, AI proposes a sprint: 5-8 tasks mapped to milestones, with time estimates and grind-work blocks scheduled.

4. **Daily Focus Queue** — Each morning, 3 ranked items. Not "do these if you feel like it" — "these are the critical path items, and here's why item 1 blocks everything else."

5. **Grind Automation** — The chat agent autonomously handles or pre-processes:
   - Email triage: categorise inbox, draft responses, flag urgent items
   - Accounting: weekly reconciliation reminders, auto-categorise transactions
   - Admin: renew tokens, check certificate expiry, update documentation

6. **Drift Detection** — If Will starts working on something not mapped to any milestone, the system asks: "This looks like a new initiative. Want to (a) add it as a goal, (b) park it for later, or (c) continue knowing it's off-plan?"

7. **Outcome Tracking** — When a milestone is marked complete, the system measures: did the metric move? Was the goal advanced? Feed this back to the next sprint.

**What it solves:**
- A (what to work on): Yes — data-driven daily queue tied to goals
- B (critical path): Yes — milestone decomposition with dependency ordering
- C (grind work): Yes — automated where possible, scheduled where not
- D (novelty drift): Yes — active detection and gentle redirection

**Effort:** ~1-2 weeks. New MongoDB collections, focus-manager script, chat agent integration, Chief of Staff agent extension.

**Limitation:** Requires Will to define goals upfront and engage with the sprint process. The system is only as good as the goals it's optimising for. Also — drift detection could feel annoying if poorly calibrated.

### Option C: Autonomous Operating System (Full Vision)

**Concept:** The AI doesn't just advise or track — it operates significant parts of the business autonomously, with Will as the decision-maker on strategy and the AI as the executor on everything else.

**How it works:**

The business has three layers:
1. **Strategy** (Will decides): What markets, what product, what pricing, what brand voice
2. **Execution** (AI executes): Pipeline ops, content generation, ad management, email, accounting, reporting
3. **Judgement calls** (AI proposes, Will approves): New features, budget changes, outreach, pricing

Today, Will operates across all three layers. Option C moves him firmly into layer 1, with AI handling layer 2 and presenting layer 3 decisions for approval.

**New capabilities required:**

| Capability | What Changes |
|-----------|-------------|
| **Autonomous email** | Chat agent checks inbox 3x/day, categorises, drafts responses, sends routine replies, escalates important ones to Will |
| **Autonomous accounting** | Weekly auto-reconciliation, expense categorisation, monthly P&L generation, tax deadline tracking |
| **Autonomous content** | Article pipeline runs without intervention — generate, fact-check, publish, promote on Facebook |
| **Autonomous ad management** | CEO Growth agent adjusts budgets, pauses underperformers, tests new creatives within safety caps |
| **Proactive outreach** | When a milestone requires "contact 3 agents", AI identifies candidates, drafts emails, queues for Will's send-approval |
| **Calendar management** | AI schedules Will's week: deep work blocks, grind blocks, review blocks |

**What it solves:**
- All four problems completely
- Also solves the meta-problem: Will's time becomes the scarcest resource, and the system optimises for it

**Effort:** 4-8 weeks across multiple systems. Significant trust-building required — autonomous email and ad management need careful guardrails.

**Limitation:** This is the vision, not the starting point. Trust must be earned incrementally. One wrong autonomous email or ad decision erodes confidence in the entire system. Also, premature autonomy without clear goals just automates randomness.

## 4. Why This Matters (Beyond Productivity)

This isn't a productivity hack. It's a structural change in how a solo founder operates.

### The Solo Founder Bottleneck

A solo founder is simultaneously CEO, CTO, marketer, accountant, and customer support. The failure mode isn't laziness — it's **context switching cost**. Every time Will shifts from pipeline debugging to email to ad review to content writing, he pays a 15-30 minute cognitive tax. With ~6 productive hours per day, that's potentially 2-3 hours lost to switching.

An AI focus system doesn't add hours to the day. It **reduces the number of decisions about what to do next** — which is where most of the switching cost lives.

### The Compound Effect of Correct Sequencing

If the critical path to first revenue is: coverage → valuation accuracy → report template → agent outreach → first sale, then every hour spent on the right step in the right order compounds. Every hour spent on something off-path (however interesting) delays everything downstream.

The difference between "busy" and "productive" is sequencing. A focus system that gets sequencing right could compress a 6-month timeline into 3 months — not by working harder, but by eliminating wasted motion.

### What Changes for Will as a Human

The four failure modes Will identified aren't just business problems. They're quality-of-life problems:
- **Not knowing what to work on** creates anxiety
- **Missing the critical path** creates the feeling of running but not moving
- **Avoiding grind work** creates guilt and a growing backlog of dread
- **Chasing novelty** creates excitement followed by disappointment when nothing ships

A well-built focus system addresses all four emotional states, not just the business outcomes. Will would end each day knowing: I worked on the right things, the boring stuff is handled, and I'm measurably closer to the goal. That's a different experience of work.

## 5. Honest Assessment of Risks

### Risk 1: Over-engineering the system instead of using it
The meta-irony: building a focus system is itself a novelty project that could distract from the actual business milestones. The system must be simple enough to build in days, not weeks.

**Mitigation:** Start with Option A or a minimal Option B. Don't build Option C until Option B is proven.

### Risk 2: Bad goals in, bad priorities out
If the goals are wrong ("get 100K visitors" when the real goal is "get 1 paying customer"), the entire system optimises for the wrong thing.

**Mitigation:** Goal-setting session with Will before building anything. Use the "first principles" test: what is the smallest thing that proves this business works?

### Risk 3: Drift detection feels like nagging
If every tangent triggers a "you're off-plan" warning, Will will start ignoring the system or resenting it.

**Mitigation:** Calibrate sensitivity. Some tangents are valuable exploration. The system should distinguish between "completely unrelated distraction" and "adjacent exploration that could feed back into a goal." Ask, don't lecture.

### Risk 4: Automation without oversight
Autonomous email or ad management that makes a mistake is worse than doing nothing — it actively damages the business.

**Mitigation:** Every autonomous action starts in "draft mode" — the system prepares, Will approves. Autonomy is earned by demonstrating accuracy over time.

### Risk 5: The system becomes another thing to maintain
If the focus system itself requires daily feeding, debugging, and upkeep, it adds load instead of removing it.

**Mitigation:** The system must be self-maintaining. If it breaks, it should fix itself or degrade gracefully (fall back to a simple daily email with priorities).

## 6. Recommended Starting Point

**Build a minimal Option B in the next session.** Here's why:

- **Option A is too passive.** It's just a better dashboard. Dashboards don't change behaviour — they inform it, and information isn't Will's bottleneck. Decision and execution are.
- **Option C is premature.** Autonomous operations require trust that hasn't been built yet. And the infrastructure for autonomous email/accounting isn't ready (email auth isn't even configured yet).
- **Option B hits the sweet spot.** It's active (proposes daily focus, detects drift, handles grind), but not autonomous (Will approves, AI executes on command). It can be built on existing systems (CEO agents, chat agent, todo manager) with relatively modest new code.

### Minimum Viable Focus System (Option B, Phase 1)

1. **Goal Registry** — 3-5 goals stored in MongoDB with target dates
2. **Milestone Map** — AI-generated dependency tree per goal, Will-reviewed
3. **Daily Focus Queue** — 3 items each morning via Telegram + chat agent context
4. **Grind Schedule** — Fixed weekly blocks: "Monday AM = email triage, Wednesday PM = accounting review"
5. **Drift Flag** — Gentle check when work doesn't map to any active milestone
6. **Weekly Review** — Friday automated summary: what shipped, what slipped, next week's sprint

### What We Don't Build Yet (Phase 2+)

- Autonomous email triage (need auth configured first)
- Autonomous ad management (need more confidence in valuation accuracy)
- Calendar integration (no calendar system exists)
- Proactive agent outreach (no prospect database)

### Data Flow

1. **Will sets goals** (via chat agent or direct DB entry)
2. **AI decomposes** into milestones with dependencies
3. **Chief of Staff** synthesises daily: goals + proposals + todos + ops → focus queue
4. **Will sees** 3 items each morning via Telegram, chat agent loads as context
5. **Will works** — chat agent tracks which focus item he's on
6. **Drift detection** — if work doesn't map to any item, gentle nudge
7. **End of day** — auto-log what was completed, update milestone progress
8. **Weekly** — sprint review, next sprint proposal, milestone health check
9. **Monthly** — goal review, re-prioritise, retire completed goals

## 8. The Human Impact Thesis

If this works — if Will consistently works on the right thing at the right time, with grind work handled or scheduled, and novelty tangents consciously chosen rather than unconsciously drifted into — the impact is:

1. **Faster to revenue.** The critical path is visible and followed. Wasted motion drops.
2. **Less anxiety.** The "am I doing the right thing?" question has a data-backed answer.
3. **Grind work actually happens.** Scheduled, tracked, partially automated. No more guilt backlog.
4. **Novelty is channelled, not suppressed.** The system doesn't ban exploration — it makes it a conscious choice with visible trade-offs.
5. **The business scales without hiring.** If AI handles execution and grind, Will's role becomes pure strategy and relationship-building — the things a founder should actually spend time on.

This is what "AI-native business" actually means. Not AI generating content or analysing data (we already do that). AI managing the cognitive load of running a business alone.
